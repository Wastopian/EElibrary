## Mission Alignment

EE Library is a **private engineering memory system for hardware teams**. The repo ships a full **part-readiness loop** (search, import, detail, admin review, export gating) plus a **project/BOM memory** track: persisted projects and revisions, CSV and XLSX BOM intake, column mapping, deterministic row matching, confirmed `project_part_usages`, explainable **BOM health** and **fleet risk**, **revision compare**, **where-used** (parts, circuit blocks, connector sets, assets), **evidence** workspace, **circuit blocks** (library, detail, instantiation, reuse signals, linked-part metric rollups), **export bundles** with download UI, **approved substitutes**, **lifecycle regression** findings, **`/connector-sets`** catalog (FUNC15), **approval batch** actions from project context (FUNC16), and a first **`/tools`** scratchpad workspace for evidence-note-backed EE calculations.

Public provider/catalog data is input. Internal engineering truth is the product: approvals, evidence, usage history, risk findings, and follow-ups—not distributor browse parity.

The next major direction is to make that engineering memory the **best tool for engineering teams**: turn a strong single-operator record into a genuinely **multi-engineer** system that is shared, governed, accountable, and concurrent—without weakening the imported ≠ approved ≠ export-ready discipline. The governance foundation (request-pipeline **audit log**, **project revision approval gate**, **controlled document revisions / ACL / redlines**, and a **vendor notebook**) is already shipped; scoped RBAC, SSO, concurrency, multi-stage ECN/ECO, and real ECAD emission are the planned arc (see **Team Collaboration and Governance** below).
